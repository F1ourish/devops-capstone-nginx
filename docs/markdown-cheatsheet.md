# Markdown Cheatsheet (минимум для документации)

## Заголовки
Пример:
# H1
## H2
### H3

## Текст
**жирный**, *курсив*, ~~зачёркнутый~~

## Списки
- пункт
  - подпункт
1. пункт
2. пункт

## Ссылки
[Текст ссылки](https://example.com)

## Оглавление и якоря (ссылки на разделы)
### Как сделать кликабельное оглавление

   1. Пишешь заголовки (`##`, `###`).

   2. В начале файла добавляешь список ссылок вида: `[Текст](#якорь)`.

Пример ссылки на заголовок:

* `[Быстрый старт](#0-быстрый-старт-90-секунд)`

### Как формируется якорь (на GitHub)

Обычно:

* текст заголовка приводится к нижнему регистру

* пробелы превращаются в `-`

* часть символов удаляется (например `/` часто “съедается”)

Если в заголовке было `origin / repo`, то якорь может получиться с двойным дефисом: `origin--repo` (из-за пробелов вокруг `/`).

Самый надёжный способ не угадывать:

* на GitHub открой файл → наведи на заголовок → нажми на значок ссылки слева → скопируй готовый `#...` и вставь в оглавление.

## Вложенные списки
### Вложенный маркированный список

Смысл: подпункты делаются отступом (обычно 2–4 пробела).
Пример структуры:

* `1. пункт`

   * `подпункт`

   * `подпункт`

### Вложенный нумерованный список

Можно делать и так:

* `1. пункт`

   * `1. подпункт`

   * `2. подпункт`

Примечание: GitHub умеет сам правильно пронумеровать, даже если ты внутри пишешь везде `1.` — это нормальная практика.

Инлайн-код (серый фон)

Чтобы выделить команду/флаг прямо внутри текста — оборачивай в одинарные бэктики:

* `git pull --rebase origin main`

* `ssh -T git@github.com`

* `systemctl status nginx`

Инлайн-код удобен для коротких фрагментов в предложении. Для многострочных примеров лучше использовать блок кода.

## Ссылки на файлы и разделы внутри репозитория
### Ссылка на файл

Пример: `[Runbook](docs/runbook.md)`

Ссылка на раздел в файле

Пример: `[Drill 1](docs/runbook.md#drill-1-nginx-остановлен)`
Важно: якорь `(#...)` должен соответствовать реальному заголовку в файле (см. раздел “Оглавление и якоря”).

## Важные заметки (выделение внимания)

Цитата/заметка оформляется через `>`:

* `> Важно: ...`

* `> Совет: ...`

* `> Осторожно: ...`

## Частые ошибки в Markdown (быстрый чек)

* “Список сломался” → проверь отступы у вложенных пунктов.

* “Оглавление не кликается” → якорь неверный; лучше скопировать ссылку на заголовок прямо на GitHub.

* “Код/разметка поехали” → проверь закрывающие бэктики и пустые строки между блоками.

## Код
Инлайн:
`systemctl status nginx`

Блок кода:
```bash
systemctl status nginx
journalctl -u nginx -n 50 --no-pager
```

Пример:

> Совет: держи картинки в `docs/img/` и называй файлы понятно.
