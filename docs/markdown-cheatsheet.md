# Markdown Cheatsheet (минимум для документации)

## Заголовки
Пример:
# H1
## H2
### H3

## Текст
**жирный**, *курсив*, ~~зачёркнутый~~

## Списки
- пункт
  - подпункт
1. пункт
2. пункт

## Ссылки
[Текст ссылки](https://example.com)

## Код
Инлайн:
`systemctl status nginx`

Блок кода:
```bash
systemctl status nginx
journalctl -u nginx -n 50 --no-pager
```

## Картинки
Если картинка лежит в репозитории (например, в `docs/img/`):

Пример:
`![Описание](img/dashboard.png)`

> Совет: держи картинки в `docs/img/` и называй файлы понятно.

## Таблицы
Пример:

| Поле |   Значение   |
|------|--------------|
| OS   | Ubuntu 24.04 |
| Svc  |    nginx     |

## Цитаты / заметки
Пример:
> Важно: сначала фиксируем симптомы, потом делаем выводы.

## Чек-листы
Пример:
- [ ] сделать X
- [x] сделать Y

## Линия-разделитель
Пример:
---

## Ссылки на файлы внутри репозитория
Пример:
`[Runbook](docs/runbook.md)`
